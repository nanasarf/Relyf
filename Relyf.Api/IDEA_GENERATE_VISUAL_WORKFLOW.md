## ?? Key Data Flows

### 1. Authentication & Authorization
```
JWT Token (from login)
    ?
Extract "sub" claim ? userId (4)
    ?
Verify user exists in app.User
    ?
If itemId provided: verify user owns item
```

### 2. Prompt Processing
```
Input promptText
    ?
Save to app.CoherePrompt (with model config)
    ?
Add system message
    ?
Send to Cohere API
```

### 3. Response Handling
```
Cohere Returns ideaText
    ?
Hash promptText with SHA256
    ?
Log to app.ApiRequestLog
    ?
Save ideaText to app.AiIdea
    ?
Return 201 Created
```

## ?? Example Flow: Real Scenario

```
1??  User (id: 4) logs in
    ? Gets JWT token

2??  User uploads item: "Old Jeans" 
    ? Item stored (id: 5)

3??  User requests upcycling ideas
    ? POST /api/ideas/generate
       {
         itemId: 5,
         promptText: "Old jeans ideas?",
         titleHint: "Jean Upcycling"
       }

4??  Server validates:
    ? JWT token is valid
    ? User (4) exists
    ? Item (5) belongs to user (4)

5??  Server saves prompt config
    ? CoherePrompt inserted (id: 15)

6??  Server calls Cohere AI
    ? Returns: "1. Denim Tote Bag... 2. Jean Jacket... 3. Denim Rug..."

7??  Server logs the request
    ? ApiRequestLog inserted

8??  Server saves the idea
    ? AiIdea inserted (id: 42)

9??  Server responds (201 Created)
    {
      ideaId: 42,
      title: "Jean Upcycling",
      ideaText: "1. Denim Tote Bag... 2. Jean Jacket... 3. Denim Rug...",
      coherePromptId: 15,
      itemId: 5,
      userId: 4
    }

?? User sees the generated ideas!
```
